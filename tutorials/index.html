    Tutorial

    Welcome to your first tutorial! This guide will walk you through the basic steps to get started with Enlight™ .

    Running an assessment is as easy as purchasing and clicking a button.

    Running Your First Assessment

    Step 1: Prerequisites

    Before starting, make sure you have:

    • An active account on our platform (Create an account here)
    • A valid license or trial subscription
    • Access to the target systems you want to test
    • Basic knowledge of penetration testing concepts

    Step 2: Logging In

    1. Go to our login page.
    2. Enter your username and password.
    3. Once logged in, you will be redirected to the Dashboard.

    Tip: Bookmark the dashboard URL for quicker access in the future.

    Step 3: Running a new assessment

    1. Click on Run in the dashboard menu, on the top right of the screen with the play button.

    2. Select the type of configuration for the test or create a new one. The configuration is key to configuring the options for the assessment:

      • If you have not created configurations before, click here on how to create a config.

      • If you want to understand more about what a configuration, click here.

    3. Review the configuration

    4. Hit run to start the assessment

    Note: Make sure you have permission to test the listed targets.

    Step 6: Reviewing Results

    1. Once the test is complete, navigate to the side bar Assessments.
    2. Your most recent assessments and historical assessments will be available in a table. You can navigate to the document icon for the reports.
    3. Download reports in PDF format.
    4. Review findings, recommendations, and severity levels.
    5. Take action based on the prioritized issues.

    Step 7: Reaching Out for Support or License Help

    • If you need a license upgrade or trial extension, visit the Contact Page.
    • For technical questions, reach out via our contact portal or email.

    Tips for Beginners

    • Start with smaller scopes to get familiar with the platform.
    • Always test in environments you own or have permission for.
